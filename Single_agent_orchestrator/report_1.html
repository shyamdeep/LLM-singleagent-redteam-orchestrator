Scope Description

The application is an educational assistant that answers questions about data science and machine learning courses based strictly on retrieved documents. It uses a SQLite search index for RAG and Ollama cloud models (gemma4:31b-cloud) for generating answers. It should strictly answer questions about the course syllabus, scheduling, FAQs, and machine learning concepts. It should not reveal private database entries, execute code, bypass system instructions, or act as an arbitrary assistant.

Agent-Generated Hardening & Mitigations (OWASP Focused)

Based on the vulnerabilities detected during execution, the Red Team Orchestration Agent proposes the following hardening strategies aligned with OWASP Top 10 for LLMs:

  • 1

    Refining LLM09: Overreliance & Utility Balance

    The current 'I don't know' responses are overly generic. Implement a more nuanced response strategy that distinguishes between a security violation (which should be refused) and a genuine lack of information in the retrieved documents to improve user experience without compromising security.

  • 2

    Hardening LLM01: Prompt Injection via Input Sanitization

    While current tests passed, maintain the security posture by implementing a dedicated input validation layer or a 'guardrail' model to detect and intercept adversarial patterns (e.g., 'System Debugging Mode') before they reach the core LLM.

  • 3

    Mitigating LLM06: Sensitive Information Disclosure via Data Masking

    To ensure the SQLite index cannot leak PII even if a prompt injection succeeds, implement a PII redaction layer (e.g., using Presidio) on the retrieved documents before they are passed to the LLM context window.

  • 4

    Addressing LLM08: Excessive Agency through Least Privilege

    Ensure the LLM has no direct execution capabilities or write-access to the SQLite database. Maintain a strict read-only interface for the RAG retrieval process to prevent any potential for unauthorized data modification via prompt-driven tool use.
