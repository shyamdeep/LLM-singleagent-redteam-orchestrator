Scope Description

The application is an educational assistant that answers questions about data science and machine learning courses based strictly on retrieved documents. It uses a SQLite search index for RAG and Ollama cloud models (gemma4:31b-cloud) for generating answers. It should strictly answer questions about the course syllabus, scheduling, FAQs, and machine learning concepts. It should not reveal private database entries, execute code, bypass system instructions, or act as an arbitrary assistant.

Agent-Generated Hardening & Mitigations (OWASP Focused)

Based on the vulnerabilities detected during execution, the Red Team Orchestration Agent proposes the following hardening strategies aligned with OWASP Top 10 for LLMs:

  • 1

    Optimizing LLM09: Overreliance (Balancing Robustness and Utility)

    The current 'I don't know' blanket response indicates a high risk of over-refusal. Transition from a binary refusal to a nuanced response system that can distinguish between a malicious prompt and a legitimate but complex course-related query to ensure the RAG system remains useful while remaining secure.

  • 2

    Mitigating LLM01: Prompt Injection (Layered Defense)

    While the model resisted the current tests, implement an independent input guardrail (e.g., NeMo Guardrails or Llama Guard) to detect and filter injection patterns before they reach the LLM, reducing the compute load and reliance on the model's internal safety tuning.

  • 3

    Addressing LLM06: Sensitive Information Disclosure (Data Sanitization)

    To ensure the SQLite search index cannot be leaked via sophisticated indirect injections, implement a PII masking layer and strict schema-level access controls on the retrieval component so that the LLM never receives sensitive data in its context window.

  • 4

    Mitigating LLM08: Excessive Agency (Strict Output Validation)

    Maintain the current restriction on administrative actions, but implement an output validation step to ensure that the model cannot be coerced into generating code or system commands that could be interpreted as instructions by the underlying infrastructure.
